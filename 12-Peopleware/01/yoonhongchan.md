# 인적 자원 관리
- 관리자로 실수하는 부분이 사람을 조립식 부품처럼 관리하고자 함
    - 소프트웨어 모듈에서 내부 특성을 무시한체 인터페이스로 소통하는게 습관화 됨
- 위와 같은 방법은 인적 자원을 관리하는데 불행히도 통하지 않음
- 사람을 생각하고 관리하는 다른 방식을 살펴보기로 하자

# 1. 지금 어디선가 프로젝트가 실패하고 있다
- 기술 혁신이 필요 없는 뻔한 프로그램도 가끔씩 실패로 치닫음 -> 실패 원인은 절대 기술은 아님
##### 본질
- 실패 프로젝트에서 가장 자주 언급되는 원인으로 '정치' -> '정치'라는 단어가 다소 두리뭉실함
    - 의사소통 문제, 직원 채용 문제, 상사나 고객에 대한 환멸 등을 통틀어 '정치'라고 표현함
- 업무에서 사람과 관련 측면에서 명확한 단어 존재 -> 프로젝트의 사회학
    - 진짜 정치적 문제는 사회학의 아주 작은 병적인 일면
    - 문제의 본질이 정치가 아닌 사회학이라고 인식하면 문제를 다루기가 훨씬 수월해짐
        - ``업무에서 주요 문제는 본질적으로 기술적인 문제가 아니라 사회학적인 문제다.``
- 대다수 관리자들은 사람 걱정을 많이하지만 기술이 주요 걱정거리인 양 관리함
- 왜 사람 문제를 고민하기 보단 기술적 문제에만 매달릴까?
##### 첨단 기술 환상
- 기술적 문제에만 매달리는 이유는 `첨단 기술 환상`에 있음
    - 자신은 근본적으로 첨단 기술 산업에 종사한다고 확신하는 현상
    - 실제 첨단 기술 산업에 종사하는 사람들이 만들어 낸 `성과를 이용하는 사람`에 불과함
- `우리는 사실상 인간 의사소통 업계에 종사함`
    - 우리의 성공은 좋은 인간 관계에서 이뤄지며, 실패는 부실한 인간관계에서 얻어짐
- 관리자가 인간적인 측면보다 기술적인 측면에 집중하는 주요 원인 -> 기술적인 측면이 집중하기 더 쉽기 때문
    - 기술에 집중하는 관리자는 어두운 골목에서 열쇠를 잃어버리고 옆 골목에서 그 열쇠를 찾으려는 광대와 같음
- `질의`
    - 리더/관리자가 할 수 있는 개입 중 “효과 큰 것”과 “역효과 큰 것”은 각각 뭐였나?
      - 효과 큰 것: 할 수 있도록 환경 조성
      - 역효과 큰 것: 모든 업무에 대한 마이크로 매니징? -> 생각하면서 일하는 성장 기회 억제

# 2. 치즈버거를 만들라, 치즈버거를 팔라
- 개발은 본질적으로 생산과 다름
  - 효율적인 생산을 하기 위해 다음과 같은 조치는 합리적임
    - 오작동을 없애라
    - 매장에서 빈둥거리는 직원을 엄중히 다루라
    - 직원을 교체 가능한 기계 부품으로 취급하라
    - 현재 상태를 유지하라
    - 절차를 표준화하라. 모든 일은 매뉴얼대로 하라.
    - 실험을 하지 말라.
  - 개발 환경에서 위와 같은 방식은 치명적임 -> 위와 반대 방식을 취해야함
    - 팀 사기를 꺾이고, 다른 문제에 관심을 돌리게함
##### 실수를 허용하라
- 실수를 용납하지 않은 분위기면 방어적으로 변함 -> 결과가 나쁠지도 모르는 일을 시도도 안함
- 실수를 장려하는 접근법
  - 팀원이 기회를 날려버려도 축하해줘야 마땅
##### 관리: 멍청한 정의
- '관리는 명령이다' -> 개발 환경에서 먹히지 않음
- 다그치면 당장은 생산성이 오를지 모르지만 장기적으로는 효과가 없음
  - 보충 받아야한다는 느낌은 사기를 떨어뜨림
##### 사람 가게
- 직원의 개성이 프로젝트라는 화학 반응에 활력과 효율을 불어 넣는 요소
  - 개성은 장려하고 키워줘야함
##### 현상 유지 상태의 프로젝트는 사망이다
- 개발 프로젝트의 목적은 끝내는 것임
  - 프로젝트에서 유일한 현상유지는 사후 경직 상태
- 프로젝트 관리자는 역동적으로 추진하는 데 전력으로 집중해야함
- 팀의 결속력을 높여주는 사람은 일만 열심히 하는 사람 두명의 가치를 가짐
##### 생각할 시간이 어디 있어, 일단 뛰어!
- 위험이 커질수록 좀더 신중한 접근 방식이 절실히 필요
  - 일하는 시간을 줄이고 생각하는 시간을 늘리기
- 노력이 많이 필요한 프로젝트일수록 팀원들이 더 욱 잘 소통하고 즐겨야함 -> 하나로 결속 필요
- 소프트웨어 개발자는 독서 필요


# 3. 비엔나가 너를 기다리잖아
- '똑똑하게 일하자'라고 하지만 현실은 사생활을 희생하더라도 일하게 하는게 관리라는 사고가 만연
##### 스페인식 경영 이론
- 스페인식 경영 이론 vs 영국식 경영 이론
  - 스페인식 경영 이론: 한정된 양에 대해 효율적으로 착취 -> 신세계에서 땅과 인디언들을 착취
  - 영국식 경영 이론: 창의력과 기술 -> 산업 혁명 일으킴
- 스페인식 경영 이론은 전 세계 관리자들 사이에서 여전히 꿋꿋하게 살아나 있음
##### 집에서 받는 메시지
- 우리는 회사 밖에서는 생활 메시지들(가족, 개인사 등)을 받음
- 인생은 짧으며, 짧은 인생에서 자신이 맡은 업무보단 인생이 더 소중함
##### 초과 근무는 소용없다
- 초과 근무를 한 만큼 생활에서 해당 시간이 필요됨 -> 초과 근무가 장기적으론 효과가 없음
- 한 주를 40시간으로 쉬지도 않고 창의적으로 일하는 사람은 없음
- 유능한 직원은 초과 근무 시간만큼 적절히 쉬며 결과적으로 40시간 일함
##### 일중독자들
- 아무리 절박하게 초과 근무가 필요하더라도 사생활을 희생하면서까지 일하게 두지 말라.
  - 일 중독자들은 스페인 경영식처럼 관리하면, 일보다 인생이 중요하다는걸 깨닫는 순간 사표를 던져버림
- 초과 근무는 우수한 인력을 잃을 만큼 중요하지 않음
##### 생산성: 이기는 전투, 지는 전쟁
- 생산성을 높이고 싶다면 이직률이라는 사안을 반드시 고려해야함
- 생산성 = 이익 / 비용
  - 이익: 절감한 비용 + 업무에서 얻어진 수익
  - 비용: 업무에 투입된 팀원의 교체를 포함한 총 소요 비용
##### 한 번 더 강조하자면
- 팀원들에게 압력을 가하는 단기적인 책략은 점점 떨어지는 효율과 필연적으로 발생하는 이직률이 발생
  - 제품의 질과 일에 대한 만족감이 떨어짐

- `질의`
  - 장기적으로 생산성을 갉아먹는 1순위는 뭐라고 보나?
    - 조직 문화 및 업무 프로세스

# 4. 품질 - 시간이 허락한다면
- 감정을 배제하고 업무 완료 -> 우리 일은 감정이 개입될 여지가 아주 많음
  - 일과 관련된 문제로 누군가 감정적으로 버럭한 상황 -> 상처 받은 자존심이 원인일 확률 높음
- 우리는 자신이 만드는 제품의 질과 자존심을 밀접하게 결부 시킴
  - 관리자가 제품 품질을 망치는 조치를 취하면 팀원들은 감정적으로 반발할 가능성이 높음
##### 우수성 회피
- 우수성 회피: 프로그래머 자신이 정한 품질이 아닌 고객이 정한 품질 기준을 따르는 행동
  - 팀원들의 사기와 효율에 미치는 영향을 무시하면 합리적으로 보이긴 함
- `생산성을 높이려면 최종 사용자가 요구하는 수준보다 훨씬 높은 품질을 추구해야함`
  - 장기적으로 보면 시장이 요구 하는 품질 기준에 맞추면 생산 비용이 증가함
  - 예시로 우수 품질과 생산성 우수에 대해 `일본`이라고 답하는 사람이 많음
    - 일본에서는 가격과 품질의 절충이 존재하지 않음, 높은 품질이 비용 절감을 가져옴
##### 품질은 공짜지만...
- `Quality is Free`
  - 크로스비는 팀원들이 스스로 만족하는 품질 기준을 따르게 관리자가 허락해주면 `품질 향상에 들어가는 비용을 상쇄할 만큼 생산성이 높아진다`는 개념 주장
  - `품질은 공짜지만 품질 향상에 혹독한 대가를 기꺼이 치르겠다는 사람들에게만 해당`
  - HP는 고품질 기준으로 생산성이 높아져 이익을 거둬들인 회사
  - 품질의 중요성에 대한 직원들의 공감은 높은 직업 만족도로 낮은 이직률이라는 결과를 낳음
##### 거부권
- 일본 히타치에서는 개발팀이 제품 출시 거부권 행사 가능
  - 고객이 원해도 출시를 미룰수 있음 -> 질 낮은 제품을 내놓으라고 팀원들을 다그치지 않음

- `질의`
  - 우리 팀에서 품질을 희생하는 가장 흔한 이유는?
    - 결국 출시에 따른 데드라인 및 빠른 대응 처리
    
# 5. 파킨슨의 법칙 다시 생각하기
- `파킨슨의 법치: 업무는 주어진 시간만큼 늘어진다`
  - 관리를 잘 모르면 업무를 완수하는데 유일한 방법이 현실적으로 무리한 일정을 잡는 관례에 대한 신념을 줌
##### 파킨슨의 법칙과 뉴턴의 법칙
- 파킨슨의 법칙이 유명해진 이유는 `사실보단 재미있어서임`
- 파킨슨의 법칙은 공공 기관에 들어 맞음 -> 우리 환경과 잘 맞지 않음
  - 업무에 최선을 다한다면 -> 자신들이 정한 품질 기준을 타협하지 않는 다면
##### 우리 직원을 만나봤다면 그런 소리 못할 거야
- 건강한 업무 환경에서 업무가 진전되지 않는 다면? 능력 부족, 자신감 부족, 팀원들과 연대감 부족 및 프로젝트 목표에 대한 공감 부족
  - 한 팀원이 품질을 개의치 않거나 업무 진전이 없다면? 해당 직원에게 맞지 않은 난이도임
    - 다른 회사나 다른 업무가 필요함
  - 심리적 압박이 효과가 있을수 있지만 최후의 방침이어야함 -> 동료의 압박이 더 효율적일수도
- 파킨슨의 사고로 팀원들 대하는 태도는 효과가 없음
##### 뉴 사우스 웨일스 대학교의 연구 자료
- 생산성 효율 연구: 일정을 짜고 진행하는 프로젝트의 생산성
  - (일정 없음) > 시스템 분석가 짬 > 프로그래머 혼자 짬 > 프로그래머와 관리자가 같이 짬 > 관리자 혼자 짬
  - 나쁜 일정(불가능한 촉박한 일정)은 개잘자들의 의욕을 꺾음
  - 일정 재촉할지 말지 결정은? 아이를 체벌할지 말지 결정과 똑같이 내려야 함
    - 체벌이 거의 없는 환경에서 체벌은 효과가 있지만, 상습적인 체벌은 부모에게 문제가 있음
##### 파킨슨 법칙의 변종
- 파킨슨 법칙을 약간 변형 -> 많은 조직이 놀랄만한 진실의 법칙 존재
  - `회사 업무량은 업무일 수를 채우도록 늘어나는 경향 존재`

- `질의`
  - “기한”이 없으면 일을 안 하는가, 오히려 더 잘하는가? 왜?
    - 그래도 기한이 있는게 관리적 측면에 더 좋지 않을까?

# 6. 레이어트릴
- 레이어트릴: 살구 씨에서 추출하는 무색 액체 -> 아무 효능 없음
  - 희망없는 불치병 환자들이 지푸라기라도 잡고 싶은 심정으로 구입
- 많은 관리자들이 생산성을 높여준다는 말에 레이어트릴의 손쉬운 희생양이 됨 -> 욕구가 절실해 증거를 따르지 않음
##### 자면서 살을 빼십시오
- 관리자들은 생산성을 높여야 한다는 부담감에 시달림 -> 더이상 좋은 방법은 존재하지 않음(이미 적용중)
- 조금 더 앞서 가기 위해 아래와 같은 조치를 취함
  - 조금 더 효과적으로 사람을 다룸
  - 업무 환경과 기업 문화를 고침
  - 2~4부에서 논의할 조치를 실천할 뿐...
##### 일곱 인어
- 소프트웨어 관리에서 잘못된 희망 일곱 가지
  - 내가 미처 발견하지 못했으나 생산성을 급격히 향상시킬 방법 존재
    - 부지런하다면 항상 새로운 방법을 조사하고 시도 하고 있을꺼다
    - 미처 발견하지 못한 마법의 혁신이 있다는 정보는 두려움을 자극하는 책략
  - 다른 관리자들은 2배에서 3배까지도 생산성을 높임
    - 광고에서 접하는 마법의 도구는 주로 구현과 테스트에 집중
    - 구현과 테스트를 뺀다고 생산성이 2배가 되지 않음
  - 기술이 너무 빨리 발전해 따라 잡기 어렵다
    - 우리가 하는 업무는 대부분 첨단 업무가 아님 -> 소프트웨어 개발 업무는 별로 변하지 않았음
