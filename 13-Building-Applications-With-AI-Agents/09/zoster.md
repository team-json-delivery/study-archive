# 9. Validation and Measurement

## 9.1 Measuring Agentic Systems

* 에이전트 시스템은 구축하기 쉬워졌지만, 성능과 품질을 엄격히 측정하고
  검증하는 것은 여전히 큰 도전 과제입니다. 측정 없이는
  시스템이 목표를 달성하는지 알 수 없으며, 변경 사항이 개선인지 파악하기
  어렵습니다. 명확한 목표와 측정 지표를 정의하고, 체계적인
  평가 프로세스를 구축해야 고성능 에이전트 설계가 가능합니다.



### 9.1.1 Measurement Is the Keystone

* 효과적인 측정은 목표와 요구 사항에 부합하는 명확하고 실행 가능한 지표를
  선정하는 것에서 시작합니다. 정확도, 응답 시간, 견고성과 같은
  정량적 지표와 사용자 만족도와 같은 정성적 지표를 결합해야 합니다.
  특히 언어 모델 기반 에이전트에서는 의도를 파악하기 위해 임베딩 기반 거리 등
  의미론적 유사도 측정법을 활용하는 것이 현대적 표준입니다.



## 9.2 Integrating Evaluation into the Development Lifecycle

* 평가는 개발의 마지막 단계가 아니라 전체 수명 주기에 통합되어야 합니다.
  성공적인 팀은 코드 병합이나 모델 업데이트 시 테스트를 자동화하여
  회귀를 방지합니다. 자동화된 평가 외에도 정기적인 샘플링과
  인간의 검토를 병행하여 자동화로 발견하기 어려운 미묘한 문제를 포착하고
  시스템의 개선을 지속적으로 유도해야 합니다.



### 9.2.1 Creating and Scaling Evaluation Sets

* 평가 세트는 현실 세계의 다양성과 엣지 케이스를 반영해야 하며, 정적인
  테스트로는 부족합니다. 입력 상태와 예상 결과를 포함하는
  구조화된 형식을 사용하면 자동화된 점수 산정이 가능합니다.
  이 세트는 직접 작성하거나 프로덕션 로그에서 추출하며, 모델을 이용해
  의도적으로 모호하거나 복잡한 사례를 생성하여 확장이 가능합니다.



## 9.3 Component Evaluation

* 단위 테스트는 에이전트 시스템의 개별 구성 요소인 도구, 계획, 메모리,
  학습 모듈의 기능을 검증하는 핵심입니다. 각 구성 요소가
  의도대로 작동해야 전체 시스템의 신뢰성을 보장할 수 있습니다.
  이는 도구의 예외 처리부터 계획의 정밀도, 메모리의 효율성 및 학습 모델의
  일반화 능력까지 광범위한 영역을 포함합니다.



### 9.3.1 Evaluating Tools

* 도구 테스트는 일반적인 '해피 패스'뿐만 아니라 드물거나 악의적인 상황을
  포함한 모든 시나리오를 망라해야 합니다. 도구는 네트워크
  조건이나 데이터 형식에 구애받지 않아야 하며, 외부 의존성이 있는 경우
  모의 객체나 시뮬레이터를 사용하여 엣지 케이스를 검증합니다.



### 9.3.2 Evaluating Planning

* 계획 모듈은 시스템의 핵심으로, 도구 호출 시퀀스와 의사결정의 논리성을
  검증해야 합니다. 도구 회상, 도구 정밀도, 매개변수 정확도
  지표를 사용하여 에이전트가 의도를 올바르게 파악하고 적절한 작업을
  수행하는지 정량적으로 측정합니다.



### 9.3.3 Evaluating Memory

* 메모리 모듈은 데이터의 정확한 저장 및 검색뿐만 아니라, 데이터 무결성과
  시간 경과에 따른 성능 효율성을 검증해야 합니다. 검색 시
  관련성 없는 정보를 반환하지 않는지 확인해야 하며, 메모리 부하가 증가할 때
  발생할 수 있는 성능 병목 현상을 미리 벤치마킹해야 합니다.



### 9.3.4 Evaluating Learning

* 학습 모듈은 확률적 특성으로 인해 테스트가 가장 복잡하며, 과적합이나
  '망각' 현상이 없는지 엄격히 감시해야 합니다. 지도 학습이나
  강화 학습 환경에서 에이전트가 새로운 데이터에 적응하면서도 성능을
  유지하는지 확인하고, 의도적으로 분산 변화를 주어 견고성을 검증합니다.



## 9.4 Holistic Evaluation

* 통합 테스트는 에이전트의 모든 구성 요소가 실제 환경에서 상호작용하는
  전체 흐름을 검증합니다.


구조화된 입력과 예상 결과를 비교하여 전체 작업 성공 여부를 평가하며,
이를 통해 미처 예측하지 못한 복합적인 오류나 emergent behavior를
발견할 수 있습니다.



### 9.4.1 Consistency

* 일관성 테스트는 LLM의 확률적이고 비결정론적 특성으로 인해 매우 중요합니다.
  입력과 출력 간의 정렬 상태를 유지하고, 다중 턴 대화 내에서 앞선 발언과
  모순되지 않는 답변을 생성하는지 검증합니다.



### 9.4.2 Coherence

* 응집력 테스트는 에이전트의 대화가 논리적 흐름을 유지하고 문맥 정보를
  적절히 사용하여 사용자가 반복해서 말하지 않게 하는지 평가합니다.
  단순한 문법적 정확성을 넘어, 작업의 목표를 향해 순차적으로 나아가는지
  지속적으로 모니터링합니다.



### 9.4.3 Hallucination

* 환각은 잘못되거나 조작된 정보를 생성하는 심각한 문제로, 이를 방지하기 위해
  검증된 데이터 소스로 출력을 접지(grounding)하는 기술이 필수적입니다.
  지식 기반에 대한 정기적인 감사와 인간의 피드백 루프를 결합하여 정보의
  정확성을 지속적으로 관리해야 합니다.



### 9.4.4 Handling Unexpected Inputs

* 에이전트는 예상치 못한 말, 오타, 악의적인 입력에도 크래시를 일으키지 않고
  우아하게 대응해야 합니다. 적대적 변형을 포함한 체계적인
  테스트를 통해 시스템이 입력을 거절하거나 에스컬레이션하는 안전한 동작을
  보이는지 검증해야 합니다.



## 9.5 Preparing for Deployment

* 배포 준비는 단순한 테스트 통과를 넘어, 시스템이 안전하고 효율적으로
  작동함을 보장하는 홀리스틱한 품질 검사 과정입니다. 자동화된
  게이트 메커니즘을 통해 회귀가 감지되거나 성능 기준을 충족하지 못하면
  배포를 차단하여 운영 위험을 최소화합니다.



# Appendix

* **에이전트 시스템(Agentic Systems)**: 특정 환경에서 도구, 메모리, 계획 기능을 활용해 작업을 수행하는 AI 시스템.
* **도구 회상(Tool Recall)**: 기대되는 모든 도구가 정확하게 호출되었는지 측정하는 지표.
* **매개변수 정확도(Parameter Accuracy)**: 도구 호출 시 정확한 인수(인자)가 전달되었는지 평가하는 지표.
* **환각(Hallucination)**: AI가 근거 없는 잘못된 정보를 사실인 것처럼 생성하는 현상.
* **일관성(Consistency)**: 유사한 입력에 대해 에이전트가 확률적인 변화에도 불구하고 논리적이며 일관된 대응을 유지하는 성질.
* **응집력(Coherence)**: 다중 턴 대화에서 맥락을 유지하고 정보의 논리적 연결을 지속하는 능력.
* **회귀(Regression)**: 새로운 수정이나 업데이트로 인해 과거에 잘 작동하던 기능이 제대로 작동하지 않는 상태.
