## Chapter 9. 검증 및 측정
### 9.1 에이전틱 시스템의 측정

**9.1.1 측정은 핵심 (Measurement Is the Keystone)**
- 모든 평가는 목표에 부합하는 명확하고 실행 가능한 지표 정의에서 출발, 이 지표가 성능 평가의 벤치마크 역할
- 정량 지표(정확도·응답시간·precision·recall)와 정성 지표(사용자 만족도)를 함께 사용
- 언어 에이전트는 정답 형태가 다양해 exact-match로는 한계가 있어 의미 유사도(BERTScore·BLEU·ROUGE)로 보완

**9.1.2 개발 라이프사이클에 평가 통합**
- 평가를 마지막에 몰아 하거나 출력을 눈으로 훑으면 진척을 착각하기 쉬우므로 자동화된 오프라인 평가를 전 단계에 통합
- 기능이 추가될 때마다 테스트도 함께 늘려, 고정 벤치마크가 아니라 확장되는 역량 전반을 측정
- 평가 세트는 "에이전트가 무엇을 해내야 하는지"를 정의하는 살아있는 명세이자 재현성·회귀 감지의 기반

**9.1.3 평가 세트 생성 및 확장**
- 좋은 평가 세트는 입력 상태와 기대 결과를 함께 정의해 자동 채점과 지표 집계를 가능케 하는 구조
- 정적 수작업 세트는 과적합·롱테일 누락 위험이 있어 수작업·프로덕션 로그 마이닝·모델 생성으로 규모 확장
- 타겟 기법으로 적대적 프롬프팅·반사실적 편집·분포 보간 활용, 표준 벤치마크(MMLU·BBH·HELM)는 맥락화용이며 도메인엔 커스텀이 필수

### 9.2 컴포넌트 평가 (단위 테스트)

**9.2.1 도구 평가**
- 정상 경로뿐 아니라 희귀·악성·잘못된 입력까지 망라
- 출력 정확성에 더해 지연·자원 소비·오류 처리까지 검증해 부하·실패 시 우아한 저하 여부 확인
- 동일 입력엔 결정적 출력이 원칙, 외부 의존성은 mock으로 재현, 수정 때마다 회귀 테스트 필수

**9.2.2 계획 능력 평가**
- 에이전트가 의도를 올바른 행동 시퀀스로 변환하는지, 즉 "무엇을 할지 아는지"를 검증
- 도구 recall(누락 여부), precision(불필요한 호출 여부), 파라미터 정확도(인자)로 측정
- 다중 품목·모순 입력 같은 엣지 케이스와 함께 재현성·일관성 확인

**9.2.3 메모리 평가**
- 저장·검색의 정확성을 우선 검증하되, 시간 경과나 다른 작업 개입 후에도 유지되는지 확인
- 관련성(구식·무관 정보 배제)과 효율성(메모리 증가 시 검색 시간·자원)을 함께 점검
- DB 장애·데이터 손상 같은 부분 실패에서도 우아한 복구 여부 확인

**9.2.4 학습 평가**
- 진짜 개선인지, 과적합이나 망각인지 구분하는 것이 관건
- 일반화: 학습한 행동이 분포 밖 시나리오에도 적용되는지를 홀드아웃·적대적 케이스로 검증
- 적응성: 입력·환경 변화 시 치명적 망각 없는 적응 여부 확인

### 9.3 총체적 평가 (통합 테스트)

**9.3.1 엔드투엔드 시나리오 성능**
- 전체 스택을 실사용에 가까운 조건으로 처음부터 끝까지 실행해 종합 지표 산출
- 좁은 테스트셋이나 단일 지표 의존은 "지표 과적합"을 유발해 실제 유용성을 놓칠 위험
- 평가를 살아있는 과정으로 다루며, 지연·처리량과 실패 시 우아한 저하까지 함께 점검

**9.3.2 일관성**
- 모델이 확률적이라 같은 요청에도 응답이 갈리므로 표현이 달라도 입력과 정렬되는지 확인
- 긴 대화에서 이전 발언과의 모순이나 주제 이탈 여부 점검
- 액터-크리틱, LLM 기반 평가, 인간 검토를 병행

**9.3.3 응집성**
- 긴 대화에서 맥락(이전 행동·사용자 선호)을 유지하며 자연스러운 흐름 이어가는지 확인
- 사용자가 같은 말을 반복하지 않도록 이전 정보 참조가 핵심
- 모순이나 간과된 의존성은 응집성 실패로 플래그

**9.3.4 할루시네이션**
- 조작·허위 정보 방지를 위해 출력을 검증된 데이터에 그라운딩(RAG)
- 콘텐츠 정확성과 데이터 품질 의존성이 핵심 축
- 인간 참여 피드백 루프로 전파 전 교정, 비용 인지 평가도 병행

**9.3.5 예기치 않은 입력**
- 오타·슬랭·외부 서비스 부분 실패에도 크래시나 유해 출력 없는 대응 필요
- 무리한 진행 대신 명확화·거절·에스컬레이션으로 처리
- 무작위 퍼징과 과거 사례 기반 체계적 엣지 케이스 탐색, 민감정보 누출 방지

### 9.4 배포 준비
- 테스트 통과를 넘어 안전·일관·효율을 종합 점검하고 구조화된 체크리스트로 전 컴포넌트 확인
- 명확한 배포 기준 수립 (예: 도구 recall 95%, 지연·가동시간 목표)
- 회귀 감지 시 승격을 막는 게이팅과 빠른 롤백 프로세스 확보

### 9.5 마무리
- 측정과 검증은 신뢰성의 근간
- 오류 분석과 다층 평가(개별 컴포넌트~전체 상호작용)로 약점 파악
- 단위 테스트와 통합 테스트로 배포 전 문제를 잡아 자신감 있는 배포
