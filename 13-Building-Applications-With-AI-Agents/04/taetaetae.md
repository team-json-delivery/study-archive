# 4장: 도구

**개요**

* **핵심 역할**: AI 에이전트가 단순 정보 검색을 넘어 외부 환경과 상호작용하고 실제 작업을 수행하도록 지원하는 필수 구성 요소.
* **필요성**: 인간 전문가가 목적에 맞는 연장을 사용하듯, 에이전트의 실제적 변화 실행을 위해 체계적인 도구 세트가 요구됨.

---

## 4.1 랭체인 기초

* **핵심 모델**: 프롬프트 처리 및 응답 생성을 담당하는 파운데이션 모델과 채팅 모델.
* **메시지 구조화**: `HumanMessage`(사용자 입력) 및 `AIMessage`(모델 응답)를 통한 대화 맥락 유지.
* **도구 연동 방식**:
* `@tool` 데코레이터: 외부 함수(도구) 정의 및 등록.
* `.bind_tools()`: 정의된 도구를 모델에 바인딩.


* **실행 흐름**: 모델의 도구 호출 결정 → 해당 함수 실행 → 결과를 대화에 병합 → 최종 응답 생성.

### 4.1.1 로컬 도구

* **특징**: 에이전트와 함께 로컬 환경에 배포되며 사전 정의된 명시적 규칙 기반으로 동작.
* **주요 목적**: 언어 모델의 근본적 약점(산술 연산, 시간대 변환 등) 보완.
* **장점**: 높은 정밀도, 예측 가능성, 신뢰성.
* **필수 메타데이터**: 명확하고 좁은 범위의 이름, 구별되는 설명, 엄격한 입출력 스키마(오작동 방지).
* **단점**: 낮은 확장성(Scalability), 코드 중복 구현(Duplication), 잦은 유지보수(Maintenance) 비용.

### 4.1.2 API 기반 도구

* **특징**: 공용·비공개 API 연동을 통한 외부 서비스(날씨, 주식, 번역 등) 접속 및 실시간 데이터 활용.
* **장점**: 모델 재학습 없이 에이전트의 작업 범위 및 기능의 획기적 확장.
* **설계 시 고려사항**:
* 강력한 보안(HTTPS, 인증) 및 개인정보 보호 규정 준수.
* API 속도 제한(Rate limit) 대비.
* 장애 발생 시 우아한 실패(Graceful failure) 및 복구 처리.



### 4.1.3 플러그인 도구

* **특징**: 주요 플랫폼(OpenAI, Claude, Gemini 등) 및 오픈소스 생태계에서 제공하는 모듈식 확장 기능.
* **장점**: 최소한의 개발 노력으로 시스템 통합 가능, 빠른 배포 및 확장성 확보.
* **단점**: 맞춤형 개발 도구(로컬/원격) 대비 세밀한 커스터마이징 및 유연성 부족 (사용 편의성과 맞춤화 간의 트레이드오프).

### 4.1.4 MCP (모델 컨텍스트 프로토콜)

* **개념**: LLM과 외부 시스템을 일관된 방식으로 연결하는 개방형 표준("AI를 위한 USB-C 포트").
* **구조**:
* **MCP 서버**: JSON-RPC 2.0 기반으로 데이터 및 서비스 노출.
* **MCP 클라이언트**: MCP 규격을 통해 내부 구조 모른 채 서버의 메서드 호출.


* **장점**: 복잡한 맞춤형 어댑터(글루 코드) 제거, 뛰어난 상호운용성 및 도구 재사용성.
* **과제**: 인증, 세밀한 접근 제어(RBAC), 페이로드 검증 등 보안 체계의 지속적 강화 필요.

### 4.1.5 상태 유지 도구

* **위험성**: DB 등 영구적 데이터 저장소 조작 시 모델의 오류나 악의적 공격으로 인한 파괴적 결과(데이터 대량 삭제 등) 초래 가능.
* **보안 원칙 (최소 권한의 원칙)**:
* 자유로운 쿼리(예: 임의 SQL 실행) 노출 금지.
* `get_user_profile` 등 좁은 범위의 단일 작업으로 권한 캡슐화.


* **필수 조치**: SQL 인젝션 방지(입력값 검증 및 바인딩), 읽기 전용 권한 부여, 이상 패턴 탐지를 위한 철저한 로깅 및 알림.

## 4.2 도구 개발 자동화

* **개념**: 파운데이션 모델의 코드 이해 능력을 활용하여 에이전트가 사용할 도구(코드)를 자체적으로 생성 및 유지보수하는 기법.

### 4.2.1 파운데이션 모델을 활용한 도구 개발

* **특징**: API 사양 제공 시 모델이 래퍼(wrapper) 코드나 헬퍼 함수 초안을 작성하고 스스로 오류를 수정.
* **장점**: 방대한 API 환경에서 수동 코딩(보일러플레이트) 시간 대폭 절감, CI/CD 파이프라인 통합 용이.
* **한계**: 보안 및 비즈니스 로직 충족 여부에 대한 인간 개발자의 최종 검토 및 기준 설정 필수.

### 4.2.2 실시간 코드 생성

* **특징**: 에이전트가 낯선 환경이나 새로운 문제에 직면 시 작동 중 실시간으로 코드를 작성·수정·실행.
* **장점**: 동적 문제 해결 및 환경 변화에 대한 극대화된 적응력과 효율성.
* **단점 및 위험성**:
* **예측 불가능성**: 반복 실행 시마다 결과와 경로가 달라져 디버깅 및 컴플라이언스 준수 난해.
* 보안 취약점(악성 코드 인젝션) 노출 위험 및 막대한 컴퓨팅 리소스 소모.

## 4.3 도구 사용 설정

* **동작 모드 제어 (`tool-choice` parameter)**:
* `auto`: 컨텍스트에 따른 모델의 자율적 도구 호출.
* `any` / `required`: 최소 1개 이상의 도구 호출 강제 (결과값이 반드시 필요할 때).
* `none`: 도구 호출 전면 차단 (제한된 출력이나 테스트 목적).


* **오류 복구 및 후처리 메커니즘**:
* **스키마 검증**: Pydantic 등을 활용한 누락 및 형식 오류 사전 차단.
* **지능형 재시도**: 일시적 오류 시 지수 백오프(Exponential backoff) 기반 재호출.
* **우아한 대체(Fallback)**: 실패 시 백업 모델/서비스 전환 또는 안전한 기본값 반환.
* **관측성(Observability)**: 프롬프트, 도구 호출, 오류, 재시도 등 모든 과정의 로깅.

## 4.4 마무리

* **요약**: 도구는 에이전트 능력 확장의 핵심. 수동 도구(정밀함), 플러그인 도구(확장성), 자동화된 도구(적응성) 등 다양한 형태를 상황에 맞게 조합해야 함.
* **핵심 결론**: 목적에 맞는 강력한 도구 세트 구축은 성공적인 에이전트 시스템 개발의 필수 조건.
* **다음 단계 (5장 예고)**: 준비된 도구들을 계획하고 순차적으로 조합하여 복잡한 작업을 수행하는 **오케스트레이션(Orchestration)** 방법론 학습.
