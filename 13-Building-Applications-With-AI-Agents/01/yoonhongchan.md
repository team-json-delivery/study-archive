# 1. 에이전트
- 자율 에이전트는 독립적으로 추론하고 의사결정과 효과적인 상호작용하는 지능형 소프트웨어
    - 컨텍스트를 해석하고 변화하는 상황에 적응
    - 최소한의 인간 감독으로도 복잡한 작업 수행

### 1.1 AI 에이전트의 정의
- 자율 에이전트란 데이터를 스스로 분석하고, 환경을 해석하며 컨텍스트에 기반한 결정을 내리는 지능형 시스템
    - 의미 있는 의사결정을 내리고 컨텍스트에 기반해 추론하며 상황에 적응
    - 에이전트 판단 기준은? 정적인 스크립트에 따르는지 or 실제 의사결정을 하는지에 달림

### 1.2 사전학습이 일으킨 혁명
- 예전에는 머신런닝 기반 애플리케이션을 만드려면 엔지니어와 데이터 사이언티스트를 채용해 데이터를 모으고 모델을 배포해야함
- 현재는 API 호출을 통해 사전학습 생성 모델을 활용 가능해짐
    - 머신러닝과 AI 적용하는 비용과 복잡성이 크게 낮아짐
- 파운데이션 모델은 자연어 이해와 생성에 강점을 보이며 에이전트의 능력을 강화
    - 자연어 이해: 사용자 입력을 해석하고 응답
    - 컨텍스트 인지 상호 작용: 긴 대화에서도 컨텍스트 유지 -> 정확도 향상
    - 구조화된 생성: 분석, 창작 작업 지원
- 정교한 오케스트레이션 프레임워크가 결합하여 외부와 상호 작용
    - 컨텍스트 기반 해석과 의사결정
    - 도구 사용
    - 적응적 계획
    - 정보 요약
    - 비정형 데이터 처리
    - 코드 생성
    - 반복 업무 자동화
    - 멀티모달 통합

### 1.3 에이전트 유형
- 에이전트를 7가지 유형으로 분류
    - 비즈니스 업무 에이전트: 사전에 정의한 워크플로를 자동화
    - 대화형 에이전트: 자연어 인터페이스로 사용자와 상호작용
    - 리서치 에이전트: 정보, 수집, 통합, 요약을 수행
    - 분석 에이전트: 구조화 데이터를 해석해 인사이트, 대시볻, 리포트를 생성
    - 개발 에이전트: 코딩 보조도구로 코드 생성
    - 도메인 특화 에이전트: 법률, 의료, 금융 등 전문 영역에 맞게 튜닝하여 전문가 수준 지원
    - 브라우저 활용 에이전트: 웹사이트 탐색, 정보 추출 등 인간 개입 없이 수행
- 에이전트의 수와 종류는 빠르게 늘고 있으며 AI 기술이 발전할수록 새로운 형태의 에이전트는 등장

### 1.4 모델 선택
- 유료 모델부터 오픈 소스 모델까지 다양한 모델 선택 가능
    - 어떤걸 선택? 만능 모델은 없음
        - 간단한 질의는 빠르고 저렴한 소형 모델
        - 복잡한 추론은 대형 모델

### 1.5 동기에서 비동기로의 전환
- 자율 에이전트는 비동기식을 전제로 설계
    - 동시에 여러 작업을 관리
    - 새로운 정보에 재빨리 적응
    - 변화하는 조건에 따라 우선순위를 동적으로 조정
- 에이전트는 단순히 속도를 높이는 것이 아니라 일하는 방식 작체를 바꿈
    - 인간을 작업 수행 -> 작업 관리자

### 1.6 활용 사례
- 책에서 아래 에이전트를 자주 인용
    - 고객 지원 에이전트
    - 금융 서비스 에이전트
    - 의료 접수, 분류 에이전트
    - IT 헬프데스크 에이전트
    - 법률 문서 검토 에이전트
    - 보안 운영센터 분석 에이전트
    - 공급망, 물류 에이전트

### 1.7 워크플로와 에이전트
- 대다수 프로젝트는 스크립트, 결정적 워크플로, 챗봇/RAG, 자율 에이전트 중 무엇을 고르냐에 따라 우아한 해법 or 복잡한 유지보수 지옥이 됨
    - 아래 기준을 고려하여 선택
        - 입력의 가변성
        - 필요한 추론 복잡도
        - 성능/컴플라이언스 제약
        - 유지보수 부담
- 스크립트(코드)를 쓰기 좋은 상황
    - 모든 입력이 완전히 예측 가능하고 모든 출력을 미리 기술할 수 있는 상황 -> 코드가 나음
- 결정적 또는 반자동 워크플로 쓰기 좋은 상황
    - 로직을 유한한 단계나 분기로 표현할 수 있고, 인간 개입이나 추가 에러 처리가 필요한지 사전에 아는 경우
- 챗봇/RAG를 쓰기 좋은 상황
    - 자연어 이해와 문서 검색 기능을 추가하지만 자율적이고 다단계적인 계획 수립까지는 하지 않음
    - 문서 기반 Q&A가 주된 목적이고, 외부 API 호출이나 의사결정 오케스트레이션이 크게 필요하지 않은 경우
- 자율 에이전트를 쓰기 좋은 상황
    - 입력 변동성이 크고, 상황에 따라 계획이 바뀌거나 지속적 학습이 필요한 경우
    - 병렬 하위 작업이 많은 환경에서 탁월
        - 비동기로 실시간 데이터에 맞춰 재우선순위화 -> '한 번에 한단계'만 수행하는 워크풀로와 RAG와 다름
- 모든 선택은 트레이드 오프가 따름
    - 코드는 저렴하고 빠르지만 경직
    - 워크플로는 통제력이 있지만 입력이 다양하면 쉽게 깨짐
    - 챗봇/RAG는 문서 Q&A는 뛰어나지만 다단계 오케스트레이션은 못함
    - 에이전트는 강력하지만 클라우드 비용과 운영부담이 큼
- 아래와 같이 결정
    - 코드: 고정적이고 결정적인 작업
    - 워크플로: 분기가 정해져 있고, 명시적 에러 처리가 필요
    - 챗봇/RAG: 코퍼스 기반 자연어 Q&A 목적
    - 자율 에이전트: 입력의 변동성이 높거나 개방형 추론, 동적 계획, 지속적 학습 필요

### 1.8 효과적인 에이전틱 시스템 구축 원칙
- 성공적인 자율 에이전트를 구축하는 5가지 원칙
    - 확장성
        - 효율적인 인프라 관리를 지원하는 알고리즘 -> 증가하는 부하와 다양한 작업을 처리
    - 모듈성
        - 명확한 인터페이스로 연결된 독립적이고 교환 가능한 구성요소로 설계 -> 유지보수성 향상
    - 지속 학습
        - 인컨텍스트 학습과 같이 경험에서 배우는 매커니즘 구축
    - 회복탄력성
        - 오류, 보안 위협, 타임 아웃등 예기치 못한 상황을 우아하게 처리하는 아키텍처를 갖춰야함
    - 미래 대비
        - 개방형 표준과 확장 가능한 인프라 중심 설계

### 1.9 에이전틱  시스템 구축을 위한 조직 전략
- 에이전트 개발 초기 단계에서는 다양한 아키텍처, 워크플로, 모델을 자유롭게 실험하도록 장려
- 벤더 종속을 피하는 것이 중요
- 효과적인 지식 공유
- 거버넌스 프레임워크는 가볍고 유연하게 유지
- 에이전틱 시스템을 중심으로 반복적인 조직 운영
    - 탐색과 표준화 사이의 역동적 규현을 유지하기 위해 지속적인 전략을 재평가

### 1.10 에이전틱 프레임워크
- 자율 에이전트 개발을 위한 다양한 프레임워크 존재
    - 랭그래프
    - 오토젠
    - 크루 AI
    - 오픈AI 에이전트 SDK
