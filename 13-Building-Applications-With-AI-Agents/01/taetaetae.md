# 1장: 에이전트

## 1.1 AI 에이전트의 정의
- 자율 에이전트 = 데이터를 분석하고, 환경을 해석하며, 맥락 기반으로 결정을 내리는 지능형 시스템
- "에이전트"라는 용어가 남용되며 의미가 희석됨 → 에이전시는 스펙트럼
- 판별 기준: 정적 스크립트 vs 진짜 의사결정
- 발전 동력: 파운데이션 모델이 사람용 텍스트뿐 아니라 구조화된 함수 호출까지 생성 가능해짐
- 에이전틱 시스템(Agentic System) = 도구 + 메모리 + 모델 + 오케스트레이션 + 인프라의 총체

## 1.2 사전학습이 일으킨 혁명
- 전통 ML은 데이터 수집·정제에 시간 대부분 소모 → 사전학습 모델은 이 단계를 건너뜀
- API 한 번 호출로 고품질 모델 사용 가능 → AI 도입 비용·복잡도 급락
- 최신 LLM(GPT-5, Claude, Llama, Gemini, DeepSeek)이 능력 범위를 더 확장
- 파운데이션 모델이 에이전트에 주는 3가지 핵심: 자연어 이해 / 맥락 인식 / 구조화된 출력 생성
- 오케스트레이션과 결합 시 가능한 것: 도구 사용, 적응형 계획, 정보 요약, 비정형 데이터 처리, 코드 생성, 루틴 자동화, 멀티모달 종합

## 1.3 에이전트 유형
- "에이전트"라는 단어 하나로 묶기엔 종류가 너무 많다 — 최소 7+2가지.

| # | 유형 | 핵심 역할 | 대표 예시 |
| --- | --- | --- | --- |
| 1 | **비즈니스-태스크** | 사전 정의된 워크플로 자동화 | UiPath, Power Automate, Zapier |
| 2 | **대화형** | 자연어 인터페이스 응대 | 챗봇, 가상 비서 |
| 3 | **리서치** | 정보 수집·종합·요약 | Perplexity, Elicit |
| 4 | **분석** | 정형 데이터 인사이트 | Power BI Copilot, Glean |
| 5 | **개발자** | 코드 생성·리팩터링 | Cursor, Windsurf, Copilot |
| 6 | **도메인 특화** | 전문 영역 지원 | Harvey(법률), Hippocratic AI(의료) |
| 7 | **브라우저 사용** | 웹 자율 탐색·행동 | (RPA와 달리 동적 적응) |
| +1 | **음성** | 종단간 음성 대화 | 고객센터, 예약 |
| +2 | **비디오** | 아바타 기반 영상 응답 | 영업, 온보딩 |

## 1.4 모델 선택
- 모델 풍요의 시대: 상용(OpenAI, Anthropic, Google, Meta, DeepSeek) + 오픈웨이트(Llama, Mistral, Gemma)
- 증류·양자화·합성 데이터로 소형 모델도 빠르게 강해지는 중
- 합리적 출발점: 선도 제공자의 최신 범용 모델 그냥 쓰기 (HELM 기준 GPT-5 mini, o4-mini, o3, Claude 4 Opus 등 상위권)
- 비용·지연 민감 작업은 작은 모델이 거의 동등한 성능을 1/N 비용으로 제공
- 트렌드: 자동 모델 라우팅 (단순 → 소형, 복잡 → 대형)
- 미래는 멀티모델 → 처음부터 유연성 있게 설계할 것

## 1.5 동기에서 비동기로의 전환
- 전통 시스템: 동기적, 단계별 대기 → 비효율
- 자율 에이전트: 병렬 처리 + 동적 우선순위 + 실시간 적응
- 일하는 방식의 전환 사례
  - 이메일 → 답장 초안 동봉 도착
  - 인보이스 → 결제정보 사전 채움
  - 티켓 → 해결 코드 + 단위 테스트 동봉
  - CS → 추천 응답·행동 제공
  - 보안 알림 → 자동 조사 + 위협 인텔 풍부화 
- 인간 역할: 태스크 실행자(Executor) → 태스크 관리자(Manager)
- 반응적(Reactive) → 능동적(Proactive)

## 1.7 워크플로와 에이전트
- 모든 문제에 에이전트가 답은 아니다 — 4단계 중 가장 단순한 해법을 선택하라.
- 4가지 선택 기준
  - 입력 가변성
  - 추론 복잡도
  - 성능/컴플라이언스 제약
  - 유지보수 부담
- 언제 코드를 써야 하는가 
  - 결정론적 입력 (예: YYYY-MM-DD HH:MM:SS 로그)
  - 밀리초 단위 지연 요구 (임베디드/실시간)
  - 의료기기·항공·금융 등 감사 가능한 결정 로직 필수 영역
- 자율 에이전트가 빛나는 순간 
  - 자유 형식 입력 (예: "노트북 배터리가 부풀어 폭발할 것 같다" 같은 CS 이메일)
  - 병렬 서브태스크 (예: SOC가 위협 인텔 + 텔레메트리 + 샌드박스 동시 수행)
  - 실시간 데이터로 동적 재계획 (예: 공급망 일정 재조정)

## 1.8 효과적인 에이전틱 시스템 구축 원칙
-  5가지 원칙과 안티패턴

| 원칙 | 핵심 | ❌ 흔한 실패 사례 |
| --- | --- | --- |
| **📈 확장성** | 분산·클라우드·병렬 처리 지원 | 분당 10건 처리 에이전트가 1,000건 트래픽에 크래시 |
| **🧩 모듈성** | 명확한 인터페이스로 컴포넌트 분리 | 도구를 하드코딩 → 작은 변경에도 전체 재배포 |
| **🔄 지속 학습** | 인컨텍스트 학습 + 피드백 통합 | 피드백 무시 → 같은 분류 오류 반복 |
| **🛡️ 회복탄력성** | 에러·타임아웃·보안 위협 대응 | 재시도/폴백 없음 → API 한 번 실패에 전체 다운 |
| **🔮 미래 대비** | 개방 표준 + 모듈식 | 한 벤더 프롬프트 형식에 락인 → 모델 전환 불가 |

## 1.9 에이전틱 시스템 구축을 위한 조직 전략
- 조직의 흔한 함정 
  - 파편화: 너무 자유로운 실험 → 중복 PoC, 미완성 프로젝트 산재
  - 조기 표준화: 너무 빠른 통일 → 창의성 억제, 벤더/프레임워크 락인
- 조직화는 반복적(iterative) — 탐색 vs. 표준화의 균형을 지속 재평가

## 1.10 에이전틱 프레임워크
- 프레임워크가 필수는 아님 — 모델 API에 직접 구축하는 팀도 많음
- 이 책의 선택: LangGraph (직관적이면서 강력)

## 1.11 마무리
- 자율 에이전트는 AI의 변혁적 발전, 높은 자율성으로 동적·복잡 작업 수행
- 다룬 것: 기초 개념 + 전통 ML 대비 발전 + 응용 사례 + 한계
- 룰 기반 ~ 인지 아키텍처까지 각 접근은 고유 강점 + 내재적 한계
- 이 책의 목표: 그 격차를 메우는 것
