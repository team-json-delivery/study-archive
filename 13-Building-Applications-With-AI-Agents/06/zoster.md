## 제6장: 지식과 기억 (Chapter 6: Knowledge and Memory)

- 에이전트가 실제 작업을 수행하려면 파운데이션 모델의 가중치에 저장된 정보 외에도 과거의 이력을 기억하고 외부의 사실적 정보를 추가로 알아야 합니다
- 지식은 문서를 가져와 검증 가능한 정보를 제공하는 것이며, 기억은 에이전트 자신의 과거 상호작용 기록을 유지하는 것을 의미합니다
- LangGraph와 같은 프레임워크를 활용하면 전체 애플리케이션 상태를 관리하면서 기억 로직을 모듈화할 수 있습니다


### 기억에 대한 기초적 접근법 (Foundational Approaches to Memory)

- 가장 단순하지만 많은 사용 사례에서 충분히 효과적인 방식입니다


**컨텍스트 윈도우 관리 (Managing Context Windows)**
- 파운데이션 모델에 한 번의 호출로 전달되는 정보(컨텍스트 윈도우)를 활용합니다
- 윈도우가 가득 차면 선입선출(FIFO) 방식으로 가장 오래된 정보부터 삭제하는 롤링 방식을 주로 사용합니다
- 구현이 쉽지만 관련성에 상관없이 과거 정보가 유실된다는 단점이 있으며, 핵심을 놓치지 않도록 중요 문맥을 프롬프트 끝부분에 배치할 필요가 있습니다


**전통적인 전체 텍스트 검색 (Traditional Full-Text Search)**
- 대규모 검색 시스템의 백본으로, 역색인을 통해 키워드를 매핑합니다
- 주로 BM25 점수화 방식(단어 빈도, 역문서 빈도, 문서 길이 정규화 활용)을 사용하여 가장 관련성 높은 문서를 상위 K개 추출합니다
- 추출된 텍스트를 프롬프트에 직접 주입하지만, 정확한 키워드가 일치하지 않으면 숨겨진 문맥이나 개념적 연결을 놓칠 수 있습니다


### 시맨틱 기억과 벡터 저장소 (Semantic Memory and Vector Stores)

**시맨틱 검색 소개 (Introduction to Semantic Search)**
- 단순한 단어 일치를 넘어 머신러닝을 활용하여 단어와 구문의 문맥, 동의어, 관계성 등 숨겨진 의미와 의도를 파악합니다
- 텍스트를 임베딩(벡터 표현)으로 변환하여 의미가 비슷한 단어들을 가깝게 배치합니다


**벡터 저장소로 시맨틱 기억 구현 (Implementing Semantic Memory with Vector Stores)**
- 생성된 임베딩을 FAISS, Annoy, VectorDB 등 고차원 벡터 처리에 최적화된 데이터베이스에 저장합니다
- 에이전트는 쿼리 임베딩을 기반으로 유사도 검색을 수행하여 의미상 적합한 맥락을 빠르게 불러옵니다


**검색 증강 생성 (Retrieval-Augmented Generation)**
- 검색 기반 방식과 생성 모델을 결합한 기술입니다
- 문서를 작은 청크로 분할하여 벡터로 인덱싱해 둔 후, 쿼리 발생 시 관련 청크를 추출하여 파운데이션 모델에 전달함으로써 문맥이 풍부하고 정확한 응답을 생성합니다


**시맨틱 경험 기억 (Semantic Experience Memory)**
- 긴 작업을 수행할 때 초기 정보가 컨텍스트 윈도우에서 밀려나는 문제를 해결합니다
- 사용자와의 과거 상호작용을 벡터화하여 검색한 뒤, 연관성 높은 상호작용을 컨텍스트 윈도우의 일부 영역에 주입하여 개인화된 동작을 유지합니다


### GraphRAG

- 정보가 흩어져 있거나 복잡한 관계를 가질 때, 다중 홉(multihop) 추론을 가능하게 하는 고급 RAG 기법입니다


**지식 그래프 사용 (Using Knowledge Graphs)**
- 엔티티(노드)와 관계(엣지)를 명시적으로 정의하여 저장하는 그래프 데이터베이스를 활용합니다
- 검색 시스템이 관련 하위 그래프나 클러스터를 추출하면, 생성 모델이 이를 종합하여 답변을 생성합니다


**지식 그래프 구축 (Building Knowledge Graphs)**
- 데이터 수집 및 전처리, 머신러닝을 통한 엔티티 및 관계 추출, 온톨로지 설계, 그래프 채우기, 통합 및 검증, 유지보수 등의 단계를 거칩니다
- 보통 주어-술어-목적어의 시맨틱 트리플 형태로 추출됩니다


**동적 지식 그래프의 가능성과 위험 (Promise and Peril of Dynamic Knowledge Graphs)**
- 100만 토큰 이상을 처리하는 긴 컨텍스트 모델이 등장했지만, 비용과 지연 시간, 특정 팩트 검색의 정확도 면에서 검색 노드를 결합하는 하이브리드 방식이 우수할 수 있습니다
- 동적 지식 그래프는 실시간 데이터를 반영하지만 유지보수의 복잡성, 리소스 집약도, 개인정보 보안 등 위험 요소가 따릅니다


**노트 필기 (Note-Taking)**
- 파운데이션 모델이 질문에 바로 답하기 전에 제공된 컨텍스트 중 중요한 부분에 대해 스스로 주석(Notes)을 작성하도록 유도하여 깊은 추론을 가능하게 합니다


### 결론 (Conclusion)

- 기억 시스템은 단순히 데이터를 저장하는 것을 넘어 에이전트가 환경 및 사용자와 상호작용하는 방식을 변혁합니다
- 컨텍스트 윈도우 관리부터 시맨틱 기억, GraphRAG 등의 시스템을 지속적으로 개선함으로써 더 지능적이고 유능한 에이전트를 개발할 수 있습니다
