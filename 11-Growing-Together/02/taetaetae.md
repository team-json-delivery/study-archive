## 달인이 되는 비결
- 단순히 반복만 한다고 되는게 아님
- 꾸준한 반복으로 달인이 되려면 적어도
  - 실력을 개선하려는 동기가 있어야 하고
  - 구체적인 피드백을 적절한 시기에 받아야 한다.
- 특정 영역에서 개인이 성취할 수 있는 최고 수준의 퍼포먼스는 경험을 오래한다고 해서 자동으로 얻을 수 있는 것은 아닙니다 - 에릭손(전문성 연구의 대가..?)

## 수십 년 동안 전문가가 안 되는 비결
- 전문성 형성에 타당성(Validity) 과 피드백(Feedback)의 중요성
  - 타당성은 측정하고자 하는 것을 얼마나 정확하게 측정했는가
  - 피드백은 자신의 행동이나 성과 결과에 대한 정보를 되돌려 받아, 이를 통해 향후 행동을 조정하고 개선하는 과정
  - 자신이 내린 직관적 판단에 대해 빨리 피드백을 받고 이를 통해 학습할 기회가 주어지는 환경을 갖추는 것이 중요
  - 소프트웨어 개발은? 전문성이 발전될 수 있거나 그렇지 못한 직업의 중간 영역
  - 전반적으로 타당성도 낮고 피드백도 낮은 편
- 타당성과 피드백을 높이기
  - 타당성을 높이려면 : 변수를 제한하고 실험을 하면서 규칙성과 인과관계를 찾으려는 노력
  - 피드백을 높이려면 : 동료나 상사, 고객에게 개발하는 프로그램에서 직접 피드백을 적극적으로 구하기

## 당신이 제자리걸음인 이유
- 의도적 수련의 양적인 부분은 간단하나 질적인 부분은 복잡
- 의도적 수련의 필수조건, 적절한 난이도
  - 실력은 현재 수준보다 약간 높은 난이도의 과제를 반복적으로 수행할 때 향상
  - 너무 쉬운 과제 : 지루함 -> 동기 저하 -> 실력 향상 X
  - 너무 어려운 과제 : 불안감 -> 동기 저하 -> 실력 향상 X
  - 적당한 난이도의 과제 : 집중력 향상 -> 동기 부여 -> 실력 향상 O
- 실력이 늘지 않는 이유
  - 불안함이나 지루함을 느끼는 때가 대부분일 경우 실력이 늘지 않는 환경에 있는 것, 익숙해지고 습관화 됨. 자기 인식도 잘 안됨
  - 어떤 노력을 하는지에 대한 자기 인식이 부족. 자기 실력에 대해 안심. 그 상태에서 익숙해짐
- 제자리 걸음에서 벗어나기
  - 너무 쉬운 상황 <-> 너무 어려운 상황에서 벗어나는 전략 
- 지루함을 느끼는 경우 : a1 실력 낮추기
  - 의도적으로 나의 역량을 낮춰보기
  - 모든걸 AI로 활용해본다..?
- 지루함을 느끼는 경우 : a2 난이도 높이기
  - 실력은 그대로 두고 난이도를 높이기
  - 요구수준을 올리기 (rps 기준 올린다거나, 코드리뷰시 더 엄격하게 한다거나, 테스트 커버리지 더 높게 요구한다거나)
  - 공식적으로 안해도 되는 업무를 자신의 의지로 추가 하는 경우
  - 남들보다 일을 좀 더 효율적/효과적으로 하기 위해 내가 직접 만들어 쓰는 나만의 도구/방법을 묻기
  - 자주 일어나는 반복 패턴을 파악하고 분석해야 하며, 부족한 시간에도 짬을 내어 도구를 고안하고 작성
  - 쉬운일을 어렵게 하느냐 -> 의도적 수련
- 불안함을 느끼는 경우 : b2 실력 높이기
  - 실력을 올려 불안함을 해소하자
  - 사회적 접근
    - 짝 프로그래밍, 도움을 부탁, 코드 리뷰 요청
  - 도구적 접근
    - 자동화 도구 활용, 라이브러리 활용
  - 내관적 접근
    - 비슷한 일을 했던 경험을 되살리기
    - 자기 효용감이 증대하면서 스스로 인식하는 자기 실력이 향상 -> 몰입 유도
- 불안함을 느끼는 경우 : b1 난이도 낮추기
  - WTSTTCPW(What's The Simplest Thing That Could Possibly Work) 적용
  - 문제를 더 작은 단위로 쪼개기
  - 쉬운것부터 먼저 하기
- 동적인 균형
  - 자신의 업무 처리 속도가 외부적으로 문제가 되지 않는 범위 내에서 난이도와 실력을 조정 해가야
  - 메타인지 전략, 알아차림이 반드시 필요
- 팀장이 할 수 있는 일
  - 팀원의 상태를 파악하고 그들이 몰입으로 가게 도와주는 것 자체가 고도의 의도적 수련이 될 수 있음
- 이소룡의 자기 혁신
  - 이겼지만 졌던 싸움
  - 만족하지 않는 삶
