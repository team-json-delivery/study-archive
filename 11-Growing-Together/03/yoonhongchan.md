# 프로그래밍 언어 배우기의 달인
- 인지적 작업 분석: 특정 절차를 처방하기보다 작업 제약을 이해하는 데 중점을 두고 복잡한 사회기술 시스템을 모델링 하기 위해 개발된 개념적 프레임워크
    - 소위 전문가가 되려는 모든 사람에게 유용한 방법
- 인지적 작업 분석을 통해 전문성을 뽑아 내기(다음 에피소드)
##### 튜토리얼을 읽을 때 뭘 만들지 생각하고 읽는다
- 적극적 읽기: 무언가를 읽을 때 구체적인 질문이나 목적을 가지고 있는 방법
    - 질의를 통해 습득을 하게 되며, 여러 언어에서 같은 프로그램을 작성하면서 언어의 차이를 쉽게 느낌
##### 공부할 때 표준 라이브러리 소스코드를 읽는다
- 프로그래밍 업무 내 읽기 시간 > 쓰기 시간
    - 좋은 코드를 읽어봐야 좋은 코드를 쓸수 있음
- 언어 습득 시 튜토리얼 외에도 틈틈이 해당 언어의 표준 라이브러리를 찾아 읽음 -> 가장 그 언어다운 코드들의 말뭉치
    - 해당 언어의 문화와 스타일을 익힐 수 있음 -> 기능적 차이보단 프로그램 작성 및 유지보수 비용을 좌우하게 됨
    - 해당 언어의 결을 배우고 그걸 따르는 것이 중요
##### 공부 중 다른 사람의 코드에 내가 필요한 기능을 추가한다
- 자신이 만들 수 있는 작고 간단한 추가 기능 만들기
    - 유용하면서도 작고 간단한걸 생각하여 몰입을 위한 난이도 조절
    - 코드 수정하고 돌려보면서 피드백을 받을 수 있음
    - 해당 언어 오픈소스 커뮤니티 교류를 통한 피드백 받기도 가능
##### 전문성을 효과적으로 뽑아내는 전문가가 되기
- 전문가들의 전문성을 뽑아내고 적용하는 것이 자신의 전문성을 빨리 높일 수 있는 방법
    - 단기간에 놀라운 효과를 냄
- `구체적인 사건을 대해 말하도록 유도`하여 전문 비결 뽑아내기
    - `프로그래밍 언어를 빨리 배우는 비결이 무엇인가요?` 라는 일반적인 질문은 유용하지 않음 -> 일반적이면서 이론적인 대답만 나옴("연습하세요!!")
    - 전문가에게 구체적인 기억들을 상기 시키기(언어를 익힌 과정을 시간대별로 짚어가기 -> 암묵적인 의사결정과 상황판단이 무엇이었는지 추출)
- 첫 걸음은 전문가를 만나는 것이고, 구체적인 사례를 듣는것

`* 질의`
- 내가 좋은 코드라고 느끼는 기준은 무엇인지?(가독성, 일관성, 추상화 수준, 주석 스타일, 테스트 유무 등)
  - 있어야 할 게 그 자리에 예측이 되는 코드
- 완전 처음 보는 코드베이스에 처음 발을 들일 때 보통 어디 부터 보는지?
  - 바텀 업 방식으로 자그만한 기능부터 보는 편

# 실수는 예방하는 것이 아니라 관리하는 것이다
- 실수에 대대 같는 느낌은?
    - 어떻게든 피해야하고 알려지면 망신이다?
    - 좋은 학습의 기회가 될수 있다?
##### 미연에 실수를 막아야 한다?
- 산불 예방 -> 산불 관리로 바뀐 예시
    - 인위적으로 억제하면 오히려 그 지역에 가연성 물질이 과도하게 축적되게 해서 결과적으로 한분 불이 나면 엄청난 규모의 불이 남
    - 자연상태에서 적절한 시기에 작은 규모의 불이 나면 큰 규모의 불은 잘 나지 않음
- 병원 안전 프로세스가 너무 신뢰할만 해 의료 사고 발생 예시
    - 표준 운영절차 도입으로 약사가 헤파린을 준비할 때 실수할 여지가 없다고 믿어, 더이상 약 투여 시 확인해야하는 것들을 신경쓰지 않음
- 중증도 보정 사망률의 차이
    - 사망률이 낮은 병원은 합병증이 벌어지는 것을 알아채고, 적절한 조치를 함
##### 두 가지의 실수 문화
- 실무 문화에는 크게 두가지 존재
    - 실수 예방
        - 행동에서 실수로 가는 경로를 차단
        - 실수를 저지르지 말라고 요구 -> 거의 불가능
        - 실수를 한 사람을 비난하고, 처벌함 -> 실수를 감추게 되고, 논의하기 꺼려지며 협력도 덜하게 됨
    - 실수 관리
        - 실수를 조기에 발견하고 빠른 조치를 취함 -> 실수는 어떻게든 할 수 밖에 없음
        - 실수가 나쁜 결과를 내기 전에 빨리 회복하도록 돕고, 실수를 공개 -> 서로 이야기하고 배우는 분위기 형성
- 실수 예방보다 관리에 가까울수록 기역의 혁신 정도가 높음 -> 수익성도 더 높음
    - 실수가 없으면 학습하지 못하기 때문 -> 학습이론의 기본
- 불확실한 상황하에서 실수는 피할 수 없음 -> 학습을 잘하려면 실수를 격려해야함

`* 질의`
- 내가 동료의 실수를 대할 때 실제로는 “실수해도 괜찮다”고 말하지만, 행동/표정/반응에서는 “왜 그런 실수를 했냐”는 느낌을 주고 있진 않은지 돌아보기
  - 몇번 실수는 괜찮은데, 실수가 반복되면 신뢰성이 뚝 떨어져서 '또 저러네..'라는 생각이 듬... 
  - 그게 태도로 나타나는거 같기도 하고(작업에 대해서 같이 연계된 부분을 디테일하게 체크함), 상대에게 내가 너를 신뢰하지 못한다고 풍기는거 같은데... 좋지 않은 현상 같은데 어떤식으로 대처를 하면 좋을지 모르겠음..
- “실수를 관리하는 문화를 강화하는 작은 실험”을 하나만 한다면, 각자 어떤 아이디어가 떠오르는지 나눠보기
  - 'back day'라는 문화가 있어서 이슈 공유를 했음

# 뛰어난 선생에 대한 미신
- 교육은 투입으로 성과를 측정 -> 얼마나 썼냐로 얼마나 잘했냐를 가늠
    - 리더십 교육을 수료 -> 실질적 도움과는 무관
- 대부분의 훈련은 6개월 정도만 지나도 효과가 거의 사라짐 -> 당시에는 만족도도 높고 굉장히 도움된다고 느끼지만 왜 효과가 없을까?
    - 여러가지 요인이 있겠지만 교사 요소에 집중해보자!
- 교사의 지식이 학업 성취도에 미치는 영향은 미미함 -> 아는 것을 온전히 가르질수 있는가?
##### 전문가가 가르쳐주는 것은 전부가 아니다
- 행동 단계, 의사결정 단계 등 과제를 수행할 때 사용 하는 지식 중 70%는 가르치지 않음
    - '교사상'을 받은 사람, 빠짐없이 가르쳐 주라는 주문을 받아도 같은 현상 -> 필요한 것의 30%만 가르쳐 놓고 자신은 다 가르쳤다고 생각
- 반복적으로 몸에 익히고 자동화되어서 결국 암묵적으로 되어버림 -> 인식을 못하게 됨
##### 인지적 작업 분석으로 극복하기
- 위와 같은 상황을 극복 하기 위해 선생과 학생이 쓰는 방법이 존재
    - 선생: 메타인지를 높이는 노력
        - "내가 이 문제를 해결할 때 어떤 과정을 거치는가" -> 자신의 머릿속을 관찰하고, 질문을 던지고 분석
        - 학생들이 이걸 배우면서 어떤 생각을 하는가를 직접 관찰하고 질문 및 분석
        - 인지적 작업 분석에 능한 것이 학원 성취도에 미치는 영향력이 큼
    - 학생: 분석을 잘하는 선생을 고르는 것이 유리한 전략
        - 메타인지를 돕기 위해 어떻게 풀었는지 인지적 과정을 선생에게 알려주는 것도 매우 효과적
        - 선생이 푼 인지적 과정 자체를 알려달라고 요청

# 나홀로 전문가에 대한 미신
- 전문가란 전문지식은 뛰어나지만 사회성이 부족한 사람일까?
- 보통 기술을 배워서 조직으로 돌아가 활용하여 성과를 내려고 하지만 다음과 같은 이유로 실패함 -> 사회적 측면으로 인해 발생
    - 상사나 동료의 지원이 없이 추가적으로 일하려 하여, 시간이 모자라 적용하지 못함
    - 팀원/팀장들 가운데 몇몇은 추가 업무를 왜 하는지 필요성을 느끼지 못하여 스트레스를 받아 부작용 발생
    - 지원해주는 임원이 없어 확대하는데 실패하고, 별난 문화로 치부되어 실행 범위 축소
    - 아웃풋이 안나온다는 이유로 상사나 협력 팀의 리더와 갈등
    - 주면 물어볼 사람도 없어, 많은 시간 소비 후 적용 포기
    - 여력이 없어 집에서 시도해보려고 하지만 가족들 이슈로 인해 포기
- 어떤 기술적 실천법이라도 현실 적용을 위해서는 사회적 자본과 기술이 필요
    - 위와 같은 지원 없이는 실패가 보장
##### 사회적 자본과 기술
- 신뢰가 깨져 있는 상태에서는 어떤 행동을 해도 악의적으로 보임 -> `신뢰 자체가 사회적 자본의 일종`
- 사회적 자본이 좋은 사람들은 통상 사회적 기술이 뛰어남 -> 신뢰 구축을 보다 잘하는 사람
##### 고독한 전문가라는 미신
- 전문가는 사회적 자본과 사회적 기술 또한 뛰어남
    - 같은 부탁을 해도 훨씬 짧은 시간 안에 타인의 도움을 얻음
- 뛰어난 소프트웨어 개발자일수록 타인과 인터렉션에 더 많은 시간을 씀
    - 동료와의 협력에 대해 중요하게 생각함
- 사회적 훈련 개선 방법은 주변 사람들과 매일 주고받는 마이크로 인터렉션에 신경 쓰는 것
    - 인사 주고 받기, 지나가는 대화, 물어보기 등 일상적이고 소소하고 빈번한 상호작용
- 기술의 도입은 기술적 요소보다 사회적 요소가 병목될 확률이 높음
    - 어떤 기술적 지식을 전달한다고 해도 그것을 사회적 맥락 속에서 가르치고 겅험하는게 중요

`* 질의`
- 반대로, 비슷한 시도를 했는데 주변에서 도와줘서 생각보다 잘 안착된 경험이 있다면?
  - 일단 내부에서 전체적인 인식 개선 필요 -> 장급들의 지시나 대화 필요
- 하루 업무 시간의 몇 %를 “타인과 인터랙션”에 쓰고 있다고 느끼는지?
  - 옆에 분 빼고는 거의 없었던거 같음... 최근 해당 책을 읽고 만나시는 분들께 일상적인 이야기를 하려고 노력
